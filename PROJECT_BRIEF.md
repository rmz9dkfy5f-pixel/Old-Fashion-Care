# Project Brief

## Project Goal

> Old Fashion Care provides a trusted, professional web presence for a care services business — connecting elderly clients and their families with quality caregivers and support services.

---

## Problem

> Elderly individuals and their families have difficulty finding and vetting trustworthy local care providers online. Most care service websites are generic, lack social proof, and make it hard for visitors to quickly understand what is offered and how to reach out.

---

## User or Operator

> **Primary:** Elderly clients and their families seeking care services.
> **Secondary:** Caregivers seeking to connect with clients or learn about available services.

---

## Core Action

> A visitor lands on the site, quickly understands the services offered, builds trust through social proof and professionalism, and contacts Old Fashion Care.

---

## Validation Question

> Can a first-time visitor find the services page, feel confident in the quality of care offered, and reach out — all in a single visit?

---

## Smallest Useful Version

> Already live: a 6-page static site with service descriptions, contact information, and basic SEO. The site is deployed on Netlify and indexed by search engines.

---

## Success Criteria

The project is initially successful when:

- [x] Core action works — visitors can browse services and find contact info.
- [x] Site is live and publicly accessible via Netlify.
- [x] Basic SEO is in place (meta tags, Open Graph, sitemap, robots.txt).
- [x] Site is mobile-responsive.
- [x] Basic documentation exists.
- [ ] Photo gallery / social proof section is added.
- [ ] On-page SEO is audited and improved across all 6 pages.

---

## Out of Scope For Now

- [ ] Online booking or scheduling system
- [ ] User accounts or client portal
- [ ] Payment processing
- [ ] CMS or content management backend
- [ ] Blog

---

## Biggest Risks

- [ ] Photos not optimized for web — could hurt page load speed and SEO
- [ ] SEO changes accidentally affecting existing search rankings
- [ ] Contact information becoming outdated on the live site

---

## Assumptions

- [ ] Netlify deploy continues to work without configuration changes
- [ ] No CMS is needed — content updates done via direct file edits
- [ ] Photos will be provided as a folder of assets ready to use
